При выполнении задач по разработке Backend и Frontend следуй этим строгим стандартам, чтобы минимизировать ошибки:

1. **Чтение контекста перед использованием (Context First)**
   - НИКОГДА не используй кастомные компоненты (например, `EntitySelector`, `ProductSelector`, `DataTable`) или API контроллеров "по памяти".
   - ПЕРЕД написанием кода ты ОБЯЗАН прочитать файл компонента или контроллера через `view_file`.
   - Проверь точные названия пропсов (например, `searchUrl` vs `searchRoute`) и методов.

2. **Полнота реализации (CRUD Policy)**
   - Если задача формулируется как "Реализовать раздел X", это автоматически означает ЧАСТИЧНЫЙ C(r)UD (Create, Update, Delete), если не указано иное.
   - Не забывай про методы `create` и `store` в контроллерах и кнопки создания в UI.
   - Всегда проверяй `routes/admin.php` на наличие всех необходимых маршрутов.

3. **Верификация сборки (Build Verification)**
   - После любых изменений во Frontend коде (JSX/JS) ты ОБЯЗАН запустить сборку (`npm run build`).
   - Если сборка упала, ты должен исправить ошибки перед тем, как рапортовать о выполнении.
   - При внесении изменений в Backend, проверяй маршруты (`php artisan route:list`) и синтаксис (`php -l`).

4. **Использование Knowledge Items**
   - Активно используй существующие Knowledge Items для проверки архитектурных решений.
   - Если ты сомневаешься в паттерне (например, как делать фильтры), сверься с аналогичными реализованными разделами (Orders, Users).
